# Excel 导入功能使用指南

## 功能概述

Excel 导入功能允许你从标准的 CMMS Excel 模板中批量导入层级树数据（Level 1-10）。

## 主要特性

✅ **自动备份** - 导入前自动创建数据库快照
✅ **数据校验** - 自动校验数据格式和层级连续性
✅ **重复检测** - 检测并标记重复节点
✅ **错误报告** - 详细的错误和警告列表
✅ **事务安全** - 使用数据库事务确保数据一致性
✅ **新增标记** - 导入的节点自动标记为 `isNew=true`

## 使用步骤

### 1. 准备 Excel 文件

确保你的 Excel 文件符合以下格式：

**必需列**：
- `Level 1 (Company)` / `Level 1 (code)`
- `Level 2 (Vessel)` / `Level 2 (code)`
- `Level 3 (System Category)` / `Level 3 (code)`
- ... 直到 `Level 10`

**可选列**：
- `Description` - 节点描述
- 其他自定义字段（cf1-cf50）

### 2. 打开导入对话框

1. 点击工具栏上的 **📥 Import Excel** 按钮
2. 导入对话框将打开

### 3. 选择文件

1. 点击 **选择 Excel 文件** 按钮
2. 选择你的 `.xlsx` 或 `.xls` 文件
3. 文件信息将显示在对话框中

### 4. 开始导入

1. 点击 **开始导入** 按钮
2. 系统将执行以下步骤：
   - 创建数据库备份
   - 读取 Excel 文件
   - 校验数据格式
   - 检测重复节点
   - 导入到数据库

### 5. 查看结果

导入完成后，对话框将显示：
- ✓ 成功导入的节点数量
- ✗ 错误列表（如果有）
- ⚠️ 警告列表（如果有）
- 备份 ID（用于恢复）

## 数据校验规则

### 1. 层级连续性
- 不能跳级（例如：Level 1 → Level 3）
- 必须从 Level 1 开始
- 每个子层级必须有父层级

### 2. 必填字段
- 每个层级至少需要 `name` 或 `code` 之一
- Level 1 和 Level 2 是必需的

### 3. 重复检测
- 相同 `code` 和 `level` 的节点被视为重复
- 重复节点会显示警告，但不会阻止导入

## Excel 模板示例

```
| Level 1 (Company) | Level 1 (code) | Level 2 (Vessel) | Level 2 (code) | Level 3 (System Category) | Level 3 (code) |
|-------------------|----------------|------------------|----------------|---------------------------|----------------|
| PTLNG             |                | FLNG             | F04            | GENERAL                   | 0              |
| PTLNG             |                | FLNG             | F04            | ELECTRICAL                | 1              |
```

## 导入后的数据

导入的节点将：
- 自动生成唯一 UUID
- 标记为 `isNew=true`（显示为绿色高亮）
- 建立正确的父子关系
- 按 `sortOrder` 排序

## 错误处理

### 常见错误

1. **"Excel 文件为空"**
   - 检查文件是否包含数据
   - 确保第一行是表头

2. **"缺少必需列"**
   - 检查表头是否包含 `Level 1 (Company)` 和 `Level 2 (Vessel)`

3. **"层级不连续"**
   - 检查数据是否跳级
   - 确保每个子层级都有父层级

4. **"节点已存在"**
   - 这是警告，不会阻止导入
   - 重复节点会被跳过

### 数据恢复

如果导入出现问题，可以使用备份 ID 恢复数据：

```typescript
import { restoreBackup } from './services/backupService';

// 使用备份 ID 恢复
await restoreBackup('backup-id-here');
```

## 性能建议

- **小文件**（< 1000 行）：导入速度很快（< 5 秒）
- **中等文件**（1000-5000 行）：可能需要 10-30 秒
- **大文件**（> 5000 行）：建议分批导入

## 技术细节

### 导入流程

```
1. 读取 Excel 文件
   ↓
2. 校验数据格式
   ↓
3. 提取节点（按层级）
   ↓
4. 去重（同批次）
   ↓
5. 检测数据库重复
   ↓
6. 创建备份
   ↓
7. 事务导入
   ↓
8. 提交/回滚
```

### 节点 ID 生成

- 临时 ID：`L{level}_{code}` （用于建立父子关系）
- 真实 ID：`crypto.randomUUID()` （存入数据库）

### 事务安全

所有导入操作都在数据库事务中执行：
- 成功：提交所有更改
- 失败：回滚所有更改

## 下一步

导入完成后，你可以：
- 在树形视图中查看导入的节点（绿色高亮）
- 编辑节点信息
- 导出为 Excel
- 打印或导出为 PDF

## 故障排除

### 导入按钮不可用
- 确保已选择文件
- 检查文件格式（.xlsx 或 .xls）

### 导入失败
- 查看错误列表
- 检查 Excel 文件格式
- 确保数据库连接正常

### 导入速度慢
- 检查文件大小
- 考虑分批导入
- 关闭其他应用程序
