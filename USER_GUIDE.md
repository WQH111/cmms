## 操作步骤

### 1. 创建新节点 (Create)

#### 方法 A: 创建根节点（Level 1）
1. **不选择任何节点**（点击空白处取消选择）
2. 点击右侧 **[+ Create Node]** 按钮
3. 填写表单：
   - Level: 1（自动设置）
   - Name: 输入节点名称（必填）
   - Code: 输入节点代码（必填）
   - Description: 输入描述（可选）
4. 点击 **[Create]** 保存

#### 方法 B: 创建子节点
1. **先选择父节点**（点击树形视图中的节点）
2. 点击右侧 **[+ Create Node]** 按钮
3. 填写表单：
   - Level: 自动设置为父节点 + 1
   - Parent Node: 显示父节点信息
   - Name: 输入节点名称（必填）
   - Code: 输入节点代码（必填）
   - Description: 输入描述（可选）
4. 点击 **[Create]** 保存

**示例**：
```
选择: L2 FLNG
点击: [+ Create Node]
填写:
  Level: 3 (自动)
  Parent: L2 FLNG (F04)
  Name: Electrical System
  Code: ES01
  Description: Main electrical distribution system
结果: 创建 L3 Electrical System 作为 FLNG 的子节点
```

### 2. 编辑节点 (Edit)

1. **选择要编辑的节点**（点击树形视图中的节点）
2. 右侧面板显示节点详细信息
3. 点击 **[✎ Edit Node]** 按钮
4. 修改表单内容：
   - Level: 不可修改（灰色）
   - Name: 可修改
   - Code: 可修改
   - Description: 可修改
5. 点击 **[Save]** 保存修改

**注意**：
- 编辑后节点会标记为 "Modified"（橙色背景）
- Level 不能修改，如需改变层级请使用拖拽功能

### 3. 删除节点 (Delete)

1. **选择要删除的节点**（点击树形视图中的节点）
2. 点击 **[× Delete Node]** 按钮
3. 确认删除对话框：
   ```
   Are you sure you want to delete "节点名称"?
   [取消] [确定]
   ```
4. 点击 **[确定]** 删除

**⚠️ 警告**：
- 删除节点会同时删除其所有子节点
- 此操作不可撤销（除非使用备份恢复）
- 建议删除前先导出备份

### 4. 拖拽移动节点 (Drag & Drop)

1. **按住节点**（鼠标左键按住）
2. **拖动到目标位置**
   - 拖到另一个节点上：成为该节点的子节点
   - 拖到根位置：成为根节点
3. **释放鼠标**完成移动

**拖拽限制**：
- ❌ 不能拖到自己
- ❌ 不能拖到自己的子节点
- ❌ 不能拖到 Level 16 节点下（最大层级限制）
- ✅ 可以在同级之间移动
- ✅ 可以跨层级移动

**视觉反馈**：
- 拖动中：节点半透明 + 蓝色虚线边框
- 可放置：目标节点绿色背景 + 绿色边框
- 不可放置：无反馈或显示禁止图标

## 快捷操作

### 批量操作流程

**场景 1: 创建多个同级节点**
```
1. 选择父节点 (L2 FLNG)
2. 点击 [+ Create Node]
3. 创建第一个子节点 (L3 System 1)
4. 再次选择父节点 (L2 FLNG)
5. 点击 [+ Create Node]

```

**场景 2: 快速编辑多个节点**
```
1. 选择节点 A
2. 点击 [✎ Edit Node]，修改并保存
3. 选择节点 B
4. 点击 [✎ Edit Node]，修改并保存
...重复
```

## 节点状态标识

### 颜色标记
- **绿色背景**: 新创建的节点 (isNew)
- **橙色背景**: 已修改的节点 (isModified)
- **蓝色背景**: 当前选中的节点
- **灰色悬停**: 鼠标悬停状态

### 徽章显示
右侧面板会显示节点状态徽章：
- 🟢 **New**: 新导入或新创建的节点
- 🟠 **Modified**: 已编辑的节点

## 常见问题

### Q1: 为什么 [+ Create Node] 按钮可用但创建的是根节点？
**A**: 因为没有选中任何节点。如果要创建子节点，必须先选中父节点。

### Q2: 为什么 [✎ Edit Node] 和 [× Delete Node] 按钮是灰色的？
**A**: 因为没有选中任何节点。点击树形视图中的节点进行选择。

### Q3: 如何取消选择节点？
**A**: 点击树形视图的空白区域，或者刷新页面。

### Q4: 删除节点后能恢复吗？
**A**: 不能直接恢复。但如果导入前创建了备份，可以通过恢复备份来找回数据。

### Q5: 编辑节点时能改变层级吗？
**A**: 不能。层级在创建时确定，编辑时不可修改。如需改变层级，使用拖拽功能。

### Q6: 创建节点时 Level 能手动修改吗？
**A**: 创建时可以手动修改 Level（1-16），但建议使用自动计算的值以保持层级关系正确。

## 数据验证规则

### 必填字段
- ✅ **Name**: 必须填写，不能为空
- ✅ **Code**: 必须填写，不能为空
- ⭕ **Description**: 可选

### 字段限制
- **Level**: 1-16 之间的整数
- **Name**: 任意文本，支持中文、英文、特殊字符
- **Code**: 任意文本，建议使用字母数字组合
- **Description**: 任意文本，最多 1000 字符

### 唯一性检查
- 系统不强制 Code 唯一性
- 允许同名节点存在
- 建议手动确保 Code 在同一层级内唯一

## 最佳实践

### 1. 创建节点前先规划
```
Level 1: 公司/项目
Level 2: 设施/船只
Level 3: 系统类别
Level 4: 系统
Level 5: 子系统
Level 6+: 设备/组件
```

### 2. 使用有意义的 Code
```
✅ 好的命名:
  - PTLNG-001
  - FLNG-F04
  - ES-001 (Electrical System)

❌ 不好的命名:
  - 123
  - abc
  - test
```

### 3. 定期备份
- 导入前自动创建备份
- 大量修改前手动导出 Excel
- 重要操作前使用 Debug DB 检查数据

### 4. 批量操作建议
- 大量创建节点：使用 Excel 导入
- 少量创建节点：使用界面手动创建
- 结构调整：使用拖拽功能
- 批量修改：导出 Excel → 修改 → 重新导入
